# Настройка в деталях

`npm run setup` кладёт дефолтный `config.json` и поднимает пульт; критерии и
источники дальше правятся в пульте галочками. Этот файл — справочник по тому,
что в пульт не вынесено: токены, доставка, расписание и тонкости фильтра.

## Критерии поиска (config.json)

- `keywords` — фразы, которые ищутся в названии вакансии. Каждое слово фразы
  матчится как префикс слова заголовка: «дизайнер» найдёт «дизайнера»,
  «дизайн лид» найдёт «Продуктовый дизайн-лидер». Русские и английские
  термины ролей взаимозаменяемы автоматически («ведущий дизайнер» найдёт
  «Lead Designer» и наоборот) — словарь в `src/lib/filter.mjs`.
- `stopwords` — если встретилось в названии, вакансия отбрасывается целиком.
  Однословные стопы работают на обоих языках через тот же словарь («аналитик»
  режет и «analyst»). Осторожно с грейдами: стоп «middle» убьёт и
  «Middle/Senior Designer», и «Middle+» — вакансии-вилки, которые могут тебе
  подходить. Нижнюю границу грейда обычно достаточно держать ключами
  (лидовые ключи просто не заматчат чистого мидла), а стопы оставить для
  тематического мусора (чужие профессии, интерьеры и т.п.).
- К постам телеграм-каналов стоп-слова не применяются (пост длинный, одно
  упоминание «джуна» не должен его топить), а ключевые слова ищутся по всему
  тексту поста.

- `grades` — допустимые грейды: `intern / junior / middle / senior / lead /
  staff / principal / head / director`. Пустой список — любые. Грейд берётся
  из метаданных источника (если он их отдаёт), иначе выводится из названия
  («Middle/Senior Designer» имеет грейды middle И senior — пройдёт, если
  senior разрешён). Вакансии без определимого грейда не режутся.
- `formats` — формат работы: `remote / hybrid / office`. Пустой — любой.
  Смотрится по метаданным и тексту; не указан у вакансии — не режется.

Чтобы после смены критериев старые вакансии пришли заново — удали `data/seen.json`.

## Telegram-бот (доставка)

1. В Telegram: [@BotFather](https://t.me/BotFather) → `/newbot` → токен в `.env`
   как `TELEGRAM_BOT_TOKEN`.
2. Напиши своему боту любое сообщение (иначе он не может писать первым).
3. `npm run setup:telegram` — покажет `TELEGRAM_CHAT_ID` для `.env`.

## hh.ru (источник)

hh закрыл анонимный доступ к API, нужен токен приложения:

1. [dev.hh.ru](https://dev.hh.ru) → «Создать приложение» (redirect uri —
   любой, например `http://localhost`).
2. Обменяй Client ID/Secret на бессрочный токен:
   ```bash
   curl -X POST https://api.hh.ru/token \
     -d "grant_type=client_credentials&client_id=<ID>&client_secret=<SECRET>"
   ```
3. `access_token` → в `.env` как `HH_TOKEN`.
4. Перезапустить пульт: `.env` читается один раз при старте процесса, поэтому
   до перезапуска кнопка «Найти вакансии» будет считать, что токена нет
   (плановый сбор запускается отдельным процессом и подхватит сам).
   ```bash
   pkill -f src/server.mjs   # launchd поднимет обратно через пару секунд
   ```
5. Включить hh галочкой в блоке «Источники» — до появления токена он показан
   там неактивным, с пометкой «нужен HH_TOKEN».

Токен получается по `client_credentials`, то есть принадлежит приложению, а не
аккаунту: доступа к резюме и откликам он не даёт. Раздавать свой другим не
стоит — лимиты запросов считаются на приложение.

hh покрывает и компании, чьи сайты закрыты бот-протекшенами (Сбер, Контур,
Озон и т.п.) — почти все публикуют вакансии туда.

## Телеграм-каналы (источник)

Читает **публичные** каналы через их веб-ленту `t.me/s/<канал>` — **без
авторизации, без API-ключа, без аккаунта**. Просто добавь каналы в
`config.json` → `telegram.channels` (по одному в строке, формат `@handle`,
`t.me/handle` или ссылка):

```json
"telegram": { "channels": ["@designhunters", "@getitchannel"] }
```

За прогон берётся ~20 последних постов канала, дальше — только новые
(курсоры по msg_id в `data/tg-cursors.json`). Из каждого поста вытаскивается
прямая ссылка-отклик (LinkedIn / job-борды / формы) и показывается в дайджесте
как «↗ отклик» рядом со ссылкой на сам пост.

Ограничение: только публичные каналы (у приватных нет веб-ленты). Если у тебя
есть **папка-инвайт** `t.me/addlist/<...>` — разверни её в список публичных
@-каналов и добавь их в конфиг. Для этого нужен разовый вход (Telethon через
[uv](https://astral.sh/uv), ключ с [my.telegram.org](https://my.telegram.org) →
*API development tools*, в `.env` как `TG_API_ID`/`TG_API_HASH`):

```bash
npm run tg:login                       # разовый вход (код придёт в Telegram)
uv run --with telethon python src/telegram/tg_fetch.py \
    chatlist "t.me/addlist/<slug>"     # покажет каналы папки; --join — подписаться
```

## Доставка: параметры

```jsonc
"delivery": {
  "channels": ["telegram", "markdown"],        // любой набор
  "markdown": { "dir": "out" },                // или путь в Obsidian vault
  "csv":      { "file": "out/vacancies.csv" }, // трекер откликов
  "webhook":  { "url": "…", "format": "slack" } // slack | discord | text (ntfy)
}
```

Если ни один из выбранных каналов не сработал — находки страховочно
сохраняются markdown'ом в `out/`, чтобы не потеряться.

## Автозапуск

`npm run setup` ставит его сам. Отдельно — командами:

```bash
npm run service install    # поставить
npm run service            # статус (+ HTTP-проверка, что пульт отвечает)
npm run service uninstall  # снять
```

Ставятся две задачи (macOS, launchd):

| Задача | Что делает |
|---|---|
| `com.sasaki-agent` | сборщик, `StartInterval` 21600 с (6 ч) |
| `com.sasaki-agent.ui` | пульт, `KeepAlive` — стартует при входе и поднимается после падения |

Пульт под супервизором следит за `src/` и при изменении кода сам выходит —
launchd тут же поднимает его с новой версией. Это нужно потому, что ES-модули
кешируются в процессе: без перезапуска старый процесс продолжал бы отдавать
прежний код после `git pull`. Запущенный вручную (`npm run ui`) так себя не
ведёт — его некому было бы поднять.

Логи: `logs/collect.log` и `logs/ui.log`.

Linux — автоустановки нет, `npm run service` напечатает готовые строки для
crontab (сборщик) и systemd-юнита пользователя (пульт).

## LinkedIn (источник)

Источник `linkedin` берёт вакансии-объявления через **публичный гостевой
эндпоинт** LinkedIn (`jobs-guest`) — то, что видит незалогиненный посетитель.
Твой аккаунт не участвует, логина нет → риска бана нет. Это объявления о
вакансиях (job postings), а не посты в ленте.

По умолчанию ничего настраивать не надо: выбранные в пульте **направления**
переводятся в короткие английские запросы (продуктовый дизайн → `product
designer`, UX/UI → `ux designer`, иллюстрация → `illustrator` и т. д.),
поиск идёт по миру.

Параметры (`config.json` → `linkedin`, все опциональны):

```jsonc
"linkedin": {
  "location": "Worldwide", // "Worldwide" | страна | город
  "sinceHours": 168,       // окно свежести (по умолчанию неделя)
  "remoteOnly": false,     // true → только удалёнка
  "keywords": ""           // своя фраза или список фраз; перебивает направления
}
```

Две особенности их поиска, из-за которых легко получить пустую выдачу:

- **оператор `OR` не поддерживается.** Строка вида
  `design lead OR head of design` цепляется за любое «head of» — приходят
  медцентры и стройка. Нужно несколько запросов — передай список:
  `"keywords": ["product designer", "design lead"]`;
- **кириллица почти не индексируется, а выдача по России мусорная.** Замер:
  `product designer` / Worldwide — 40 релевантных из 40, `дизайнер` / Russia —
  0 из 10. Поэтому источник полезен прежде всего для международных и удалённых
  вакансий.

Поиск LinkedIn нестрогий (по одному слову тянет и смежное) — лишнее отсекает
общий фильтр по названию, так что грейды и стоп-слова работают и здесь.
LinkedIn чувствителен к частоте запросов: при `429` увеличь интервал крона.

**Посты в ленте** (персональные «ищу дизайн-лида» от нанимающих) гостевой
эндпоинт не отдаёт — их по-прежнему можно собрать полуавтоматом: попроси
Claude (с расширением Claude in Chrome) пройтись по твоей залогиненной ленте
(`linkedin.com/search/results/content/`, сортировка recent).
